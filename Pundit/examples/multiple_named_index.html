Multiple named contents: a demonstration

This demo proves how Pundit is able to stick annotations to content rather than to pages, when content is encapsulated or boxed inside a named content.

When an annotation is created on the index page, or on the child pages, it is hooked to the closest named content. This permits Pundit to visualize them in each of the sub pages, independently.

For the demonstration's sake, to each named content is added an evident dashed border.
